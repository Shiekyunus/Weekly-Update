# ---- Process Utilization ----
top -> real-time dynamic view of running processes and system resource usage
htop -> interactive and more user-friendly version of top (requires installation)
top -u user_name -> to see processes running by a specific user
ps aux --sort=-%cpu -> to list processes sorted by CPU usage (highest first)
ps aux --sort=-%mem -> to list processes sorted by memory usage (highest first)
pidof process_name -> to get the PID of a running process by name
pgrep process_name -> to search for a process by name and return its PID
kill PID -> to terminate a process by its PID
kill -9 PID -> to forcefully kill a process by its PID
killall process_name -> to kill all processes with the given name
pkill process_name -> to kill a process by name
nice -n value process_name -> to start a process with a given priority (nice value: -20 highest to 19 lowest)
renice -n value -p PID -> to change the priority of an already running process
vmstat -> to report virtual memory statistics including processes, memory, paging, I/O, and CPU
iostat -> to report CPU and I/O statistics for devices and partitions
free -h -> to display the amount of free and used memory in the system in human-readable format
uptime -> to show how long the system has been running and the load average
lsof -> to list all open files and the processes that opened them
lsof -p PID -> to list open files for a specific process

# ---- Signals & Job Control ----

kill -SIGSTOP PID -> to pause/suspend a process
kill -SIGCONT PID -> to resume a paused process
kill -SIGTERM PID -> to gracefully terminate a process (default kill signal)
kill -SIGHUP PID -> to reload/restart a process without killing it
Ctrl + Z -> to suspend a foreground process and send it to the background
Ctrl + C -> to terminate a running foreground process
Ctrl + D -> to send end-of-file signal and exit the shell
jobs -> to list all jobs running in the background or stopped
wait PID -> to wait for a process to finish before continuing

# ---- Logging ----
/var/log/ -> main directory where all system log files are stored

/var/log/messages -> general system activity logs (CentOS/RHEL)
/var/log/auth.log -> logs related to authentication and authorization (Ubuntu)

journalctl -> to view logs collected by systemd's journal

logrotate -> utility to manage and rotate log files automatically

# ---- Cron Jobs ----
crontab -e -> to edit the cron jobs for the current user
crontab -l -> to list all cron jobs for the current user
crontab -r -> to remove all cron jobs for the current user
crontab -u user_name -e -> to edit cron jobs for a specific user (as root)
crontab -u user_name -l -> to list cron jobs for a specific user (as root)
/etc/crontab -> system-wide cron job file
/etc/cron.d/ -> directory for additional cron job files
/etc/cron.daily/ -> scripts placed here run daily automatically
/etc/cron.weekly/ -> scripts placed here run weekly automatically
/etc/cron.monthly/ -> scripts placed here run monthly automatically
/etc/cron.hourly/ -> scripts placed here run hourly automatically
cron syntax: * * * * * command -> minute(0-59) hour(0-23) day(1-31) month(1-12) weekday(0-7) command
example: 0 2 * * * /backup.sh -> runs backup.sh every day at 2:00 AM
example: */5 * * * * /script.sh -> runs script.sh every 5 minutes
example: 0 9 * * 1 /script.sh -> runs script.sh every Monday at 9:00 AM
systemctl status cron -> to check if the cron service is running (Ubuntu)
systemctl start cron -> to start the cron service
systemctl enable cron -> to enable cron to start at boot

# ---- File Permissions ----
ls -l -> to view file permissions in long format
chmod 777 file_name -> to give read, write and execute permission to all (owner, group, others)
chmod 755 file_name -> owner has full permissions, group and others have read and execute
chmod 644 file_name -> owner has read and write, group and others have read only
chmod u+x file_name -> to add execute permission for the owner (user)
chmod g-w file_name -> to remove write permission from the group
chmod o+r file_name -> to add read permission for others
chown user_name file_name -> to change the owner of a file
chown user_name:group_name file_name -> to change both owner and group of a file
chgrp group_name file_name -> to change the group ownership of a file
